5-Minute Introduction for Beginners

Goal: Get a high-level understanding of DevOps in 5 minutes, then dive deeper into topics that interest you.

What is DevOps?

DevOps combines development (building code) and operations (running code). Instead of throwing code over a wall to another team, everyone works together to build, test, deploy, and monitor software.

The DevOps Pipeline (10,000 ft View)

Step 1: Developer Commits Code
You finish a feature and push code to GitHub/Azure DevOps. Automated pipeline triggers immediately.
Step 2: Automated Tests Run
System compiles code, runs 100+ unit tests, integration tests. If any fail, you get notified instantly. If all pass → move to step 3.
Step 3: Deploy to Staging
Code is deployed to a test environment that looks exactly like production. More tests run here (functional tests, security scans).
Step 4: Deploy to Production
If all tests pass, code is deployed to production (live environment). Using a blue-green deployment, users see zero downtime.
Step 5: Monitor
Application runs live. Automated monitoring watches error rates, performance, and user experience. If issues arise, alerts trigger instantly.

Key DevOps Tools

Version Control (Git)

Problem: How do 10 developers work on the same codebase?
Solution: Git tracks every change, merges code safely, prevents conflicts.

Build & Test (CI)

Problem: Testing manually takes forever.
Solution: Automated tests run on every commit (within 5 minutes).

Infrastructure as Code (Bicep)

Problem: Deploying servers/databases manually is error-prone.
Solution: Define infrastructure in code files (version-controlled, repeatable).

Deployment (Blue-Green)

Problem: Deploying breaks the app for 5 minutes = angry users.
Solution: Run 2 production copies, switch traffic between them (zero downtime).

The Interview Angle

Interviewers want to hear that you understand:

  1. Version Control: How to use git branches, create pull requests, and review code
  2. CI/CD: How automated pipelines work (build → test → deploy)
  3. Infrastructure: How servers/databases are created and scaled
  4. Reliability: How to deploy without downtime and rollback if issues arise
  5. Monitoring: How to spot problems in production instantly

Common Interview Questions

Q: "Describe your CI/CD pipeline"

Answer: Code commits trigger a pipeline with 3 stages: Build (compile, unit tests in 5 min), Test (integration tests with emulators, 10 min), Deploy (blue-green slot swap if tests pass, zero downtime). If any stage fails, pipeline stops and dev is notified.

Q: "How do you handle git branches?"

Answer: We use feature branches for each feature/bug. Create branch, make changes, push, create PR, require 2 code reviewers, tests must pass, then merge to main. Main is always deployable.

Q: "What happens if a deployment breaks production?"

Answer: We swap back the blue-green slots in <1 second. Old version goes live immediately. Then we investigate, fix, and redeploy when ready.
