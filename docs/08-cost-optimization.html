Cost Optimization

Reducing Cloud Spending

Quick Summary: Right-size resources, use reserved instances for predictable workloads, optimize storage tiers, monitor waste, and use spot instances for non-critical jobs.

Major Cost Drivers

ServiceCost ImpactHow to Reduce
App Service20-30% of totalRight-size SKUs, use B-tier for dev
Cosmos DB20-40% of totalShared throughput, on-demand pricing
SQL Database15-25% of totalReserved instances, serverless option
Data Transfer5-15% of totalUse private endpoints, CDN caching
Storage5-10% of totalArchive old data, delete unused blobs

Right-Sizing Strategy

App Service Tiers

Dev / Staging:  B1 ($10/month)
Prod Shared:    P1V2 ($100/month)
Prod High:      P2V2 ($200/month)
Prod Premium:   I1 ($500+/month)

Key Rule: Use B-tier for dev. Use P-tier only when you need scale. Measure actual usage before upgrading.

Reserved Instances

SQL Database (predictable):
  Standard: $800/month as-needed
  With 1-year reserved: $500/month (37% savings)

Timeline: Save 6-7 months to pay for the commitment

Cosmos DB Cost Control

  • Shared Throughput: One database RU/s shared across collections (not per-collection)
  • On-Demand: No commitment, pay per request (good for dev)
  • Monitoring: Set budget alerts. Alert at $500/month.

Storage Optimization

Hot Tier (frequently accessed):    $0.018 per GB/month
Cool Tier (accessed <30 days):      $0.010 per GB/month
Archive Tier (accessed <1/year):    $0.004 per GB/month

Rule: Move old backups to Archive after 30 days

Interview Answer

Q: "How do you manage cloud costs?"

We right-size resources to actual usage, not worst-case. Dev uses B1 App Services, prod uses P1V2. We use reserved instances for predictable workloads (37% savings). Cosmos DB uses shared throughput. Storage tiers: hot for current data, cool for backups, archive for historical. We monitor monthly, alert at budgets, and review under-utilized resources monthly. Last quarter we reduced costs 20% without impacting performance.
