Testing Strategy

Interview-Ready & Learning Guide

Quick Summary: Four-tier testing strategy: Unit (fast, isolated) → Integration → Functional (per-env) → Production monitoring.

Testing Pyramid

        Manual Testing & Monitoring (Production)
       /                                        \
      /    Functional Tests (1-2 hours, per env)  \
     /                                              \
    /       Integration Tests (10-15 min, emulators) \
   /                                                  \
  /               Unit Tests (1-2 min, isolated, mocked)
/________________________________________________________________\

Why this shape?

  • Unit tests (bottom, widest): Fast, isolated, lots of them
  • Integration tests (middle): Fewer, slower, test components together
  • Functional tests (narrow): Deployed app, full flow, prod-like
  • Production (top, minimal): Real users, real data, real impact

Unit Tests

Test individual methods in isolation, fast feedback.

Tools: MSTest, Moq, FluentAssertions

[TestMethod]
public async Task SendNotification_WithValidMessage_ReturnsSuccess()
{
    var notification = new Notification { Text = "Hello" };
    var result = await _service.SendNotificationAsync(notification);
    Assert.IsTrue(result);
}

Integration Tests

Test components working together using emulators.

[TestMethod]
public async Task ProcessMessage_PublishesToServiceBus()
{
    await _service.ProcessMessageAsync(message);
    var stored = await _cosmosDb.GetMessageAsync(message.Id);
    Assert.IsNotNull(stored);
}

Functional Tests

Test full deployed application end-to-end.

Run after code deployed to staging slot. If tests pass → slot swap. If tests fail → swap blocked.

Interview Answer

Q: "Describe your testing strategy"

We use a testing pyramid: Unit tests at the base (100+ per domain, <100ms each), integration tests in the middle (20-50, using emulators), functional tests above (deployed app, full flow), and production monitoring at the top. Each layer must pass before proceeding. This ensures quality at all levels while keeping CI/CD fast.
