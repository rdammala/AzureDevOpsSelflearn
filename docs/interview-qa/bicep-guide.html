Deploy Azure resources with Bicep templates

📌 Section 1: What is Bicep?

Bicep is a domain-specific language (DSL) for Azure Resource Manager (ARM) templates. It simplifies ARM template syntax and provides better maintainability.

Bicep Benefits:
✓ Simpler syntax than ARM JSON
✓ Strong type validation
✓ Modular architecture with reusable modules
✓ Automatic dependency management
✓ Seamless ARM template integration

📌 Section 2: Bicep File Structure

metadata description = 'Deploy web app'

@minLength(3)
@maxLength(24)
param storageName string

param location string = resourceGroup().location

param environment string = 'dev'

var resourceTags = {
environment: environment
created: utcNow('u')
}

resource storageAccount 'Microsoft.Storage/storageAccounts@2021-04-01' = {
name: storageName
location: location
sku: {
name: 'Standard_LRS'
}
kind: 'StorageV2'
properties: {
accessTier: 'Hot'
}
tags: resourceTags
}

📌 Section 3: Parameters and Variables

Parameters: Input values (from CLI or parameter files)

@description('Name of the resource')
@minLength(1)
param resourceName string

@allowed(['dev', 'staging', 'prod'])
param environment string = 'dev'

var resourceNameFinal = '${resourceName}-${environment}'
Parameter Files: Use .bicepparam files for environment-specific values without duplication.

📌 Section 5: Validation and Testing

// PowerShell validation
az bicep build --file main.bicep

// Validate template
az deployment group validate \
--resource-group myRg \
--template-file main.bicep \
--parameters params.bicepparam
What-If Deployment: Preview changes before deploying with --what-if flag.

📌 Section 6: Deployment Strategies

  1. Complete Mode: Delete unspecified resources
  2. Incremental Mode: Update only specified resources (default)
  3. Validation Mode: Check syntax without deploying
  4. Staged Deployments: Use multiple templates for layered architecture
Caution: Complete mode can delete production resources. Use with care!

📌 Section 7: Best Practices

  • ✓ Use parameter files for environment configs
  • ✓ Create reusable modules for common patterns
  • ✓ Add metadata and descriptions to all parameters
  • ✓ Validate templates before deployment
  • ✓ Use version control for all Bicep files
