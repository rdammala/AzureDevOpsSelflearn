☸️ Kubernetes: Container Orchestration

Deploy and manage containerized applications at scale

📌 Section 1: Kubernetes Architecture

Kubernetes automates deployment, scaling, and management of containerized applications. It abstracts underlying infrastructure and provides a declarative API.

K8s Benefits:
✓ Automatic scaling and load balancing
✓ Self-healing and rolling updates
✓ Storage orchestration
✓ Multi-cloud and hybrid support
✓ Declarative configuration

📌 Section 3: Services and Networking

apiVersion: v1
kind: Service
metadata:
name: myapp-service
spec:
type: LoadBalancer
ports:
- port: 80
targetPort: 8080
selector:
app: myapp

---

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
name: myapp-ingress
spec:
rules:
- host: myapp.example.com
http:
paths:
- path: /
pathType: Prefix
backend:
service:
name: myapp-service
port:
number: 80
Service Types: ClusterIP (internal), NodePort (external port), LoadBalancer (cloud provider).

📌 Section 4: ConfigMaps and Secrets

apiVersion: v1
kind: ConfigMap
metadata:
name: app-config

---

apiVersion: v1

metadata:
name: app-secrets
type: Opaque

password: "mysecretpassword"
api_key: "secret-api-key"

---

# Use in Pod
spec:
containers:
- name: app
image: myapp:1.0
envFrom:
- configMapRef:
name: app-config
- secretRef:
name: app-secrets

📌 Section 5: StatefulSets and Storage

apiVersion: apps/v1
kind: StatefulSet
metadata:
name: mysql
spec:
serviceName: mysql
replicas: 1
selector:
matchLabels:
app: mysql
template:
metadata:
labels:
app: mysql
spec:
containers:
- name: mysql
image: mysql:8.0
volumeMounts:
- name: mysql-storage
mountPath: /var/lib/mysql
volumeClaimTemplates:
- metadata:
name: mysql-storage
spec:
accessModes: [ "ReadWriteOnce" ]
resources:
requests:
storage: 10Gi

📌 Section 6: kubectl Commands

  1. Deploy: kubectl apply -f deployment.yaml
  2. View Status: kubectl get pods, kubectl describe pod myapp
  3. Logs: kubectl logs myapp-pod
  4. Port Forward: kubectl port-forward svc/myapp 8080:80
  5. Scale: kubectl scale deployment myapp --replicas=5
Caution: kubectl delete is permanent. Use carefully in production!

📌 Section 7: Best Practices

  • ✓ Use Deployments for stateless apps, StatefulSets for stateful
  • ✓ Set resource requests and limits
  • ✓ Use health checks (liveness/readiness probes)
  • ✓ Implement network policies for security
  • ✓ Version your images and use semantic versioning
