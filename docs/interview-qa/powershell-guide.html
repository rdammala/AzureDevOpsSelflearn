🔌 PowerShell for DevOps Automation

Automate Azure infrastructure and deployment tasks

📌 Section 1: PowerShell Fundamentals

PowerShell is a task automation and configuration management framework. It's essential for DevOps engineers managing Azure infrastructure.

PowerShell Advantages:
✓ Built-in Azure module support
✓ Object-oriented pipeline architecture
✓ Cross-platform (Windows, Linux, macOS)
✓ Rich error handling and logging
✓ Excellent for automation and scripting

📌 Section 2: Azure Authentication

# Connect to Azure
Connect-AzAccount

# Connect to specific tenant
Connect-AzAccount -Tenant 'tenant-id'

# List subscriptions
Get-AzSubscription

# Select subscription
Select-AzSubscription -SubscriptionId 'subscription-id'

# Get current context
Get-AzContext

📌 Section 3: Resource Management

# Create resource group
New-AzResourceGroup -Name 'myRg' -Location 'East US'

# List resource groups
Get-AzResourceGroup

# Get resources in group
Get-AzResource -ResourceGroupName 'myRg'

# Remove resource group
Remove-AzResourceGroup -Name 'myRg' -Force
Use hashtables for properties: Improves readability and maintainability of complex resource configurations.

📌 Section 4: Functions and Error Handling

function Deploy-WebApp {
[CmdletBinding()]
param(
[Parameter(Mandatory=$true)]
[string]$ResourceGroup,

[Parameter(Mandatory=$true)]
[string]$AppName
)

try {
Write-Host "Deploying $AppName..."
New-AzWebApp -ResourceGroupName $ResourceGroup -Name $AppName
Write-Host "Success!" -ForegroundColor Green
}
catch {
Write-Error "Deployment failed: $_"
throw
}
finally {
Write-Host "Deployment attempt completed"
}
}

📌 Section 5: Loops and Collections

# ForEach loop
$vms = Get-AzVM -ResourceGroupName 'myRg'
foreach ($vm in $vms) {
Write-Host "Restarting $($vm.Name)"
Restart-AzVM -ResourceGroupName 'myRg' -Name $vm.Name
}

# Pipeline pattern (preferred)
Get-AzVM -ResourceGroupName 'myRg' | ForEach-Object {
Restart-AzVM -ResourceGroupName 'myRg' -Name $_.Name
}

📌 Section 6: Scheduling and Automation

  1. Task Scheduler: Schedule PowerShell scripts on Windows
  2. Azure Automation: Run runbooks in Azure cloud
  3. Webhooks: Trigger scripts from external events
  4. Time-based triggers: Execute on schedule or when conditions met
Important: Always test scripts in non-production first and implement proper logging.

📌 Section 7: Best Practices

  • ✓ Always use -WhatIf parameter to preview changes
  • ✓ Implement proper error handling with try-catch
  • ✓ Use parameter validation decorators
  • ✓ Write reusable functions with clear parameters
  • ✓ Log all operations for audit trails
