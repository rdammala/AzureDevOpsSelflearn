🔌 PowerShell: Azure Automation

Automate Azure infrastructure with PowerShell

📌 Section 1: PowerShell Fundamentals

PowerShell is a task automation and configuration management framework built on .NET. Modern DevOps uses PowerShell 7+ (cross-platform).

PowerShell Basics:
✓ Object-oriented pipeline model
✓ Rich formatting and filtering
✓ Azure PowerShell module integration
✓ Cross-platform support (Core)
✓ Strong error handling capabilities

📌 Section 2: Authentication

# Modern authentication (best practice) Connect-AzAccount -Subscription "MySubscription" # List all subscriptions Get-AzSubscription # Set active subscription Set-AzContext -SubscriptionId "sub-id" # Disconnect when done Disconnect-AzAccount
Managed Identity: Use managed identity in Azure Functions instead of explicit authentication.

📌 Section 3: Resource Management

# Create resource group New-AzResourceGroup -Name "myRg" -Location "eastus" # Create storage account New-AzStorageAccount -ResourceGroupName "myRg" \ -Name "mystg" \ -SkuName "Standard_LRS" \ -Location "eastus" # List resources Get-AzResource -ResourceGroupName "myRg"

📌 Section 4: Functions and Modules

function Deploy-Application { [CmdletBinding()] param( [Parameter(Mandatory=$true)] [string]$AppName, [string]$Environment = 'dev' ) Write-Host "Deploying $AppName to $Environment" # deployment logic } # Export function Export-ModuleMember -Function Deploy-Application
Parameter Validation: Always use [Parameter()] attributes for robust scripts.

📌 Section 5: Loops and Control Flow

# ForEach loop $resources | ForEach-Object { Write-Host "Processing: $_.Name" } # While loop $count = 0 while ($count -lt 5) { Write-Host "Count: $count" $count++ } # Conditional logic if ($status -eq 'running') { Stop-AzVM -Name $vmName }

📌 Section 6: Scheduling and Automation

  • Scheduled Tasks: Use Windows Task Scheduler on local machines
  • Azure Automation: Runbooks for cloud-native scheduling
  • Cron Jobs: Use cron on Linux/macOS
  • CI/CD Pipelines: Trigger scripts from Azure Pipelines
Best Practice: Store scripts in version control and use parameters instead of hardcoded values.

📌 Section 7: Best Practices

  • ✓ Use Set-StrictMode for safer scripts
  • ✓ Implement proper error handling with try-catch
  • ✓ Use meaningful variable names and comments
  • ✓ Avoid dangerous operations without confirmation
  • ✓ Log all significant actions for troubleshooting
