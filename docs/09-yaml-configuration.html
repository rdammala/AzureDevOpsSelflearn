YAML Configuration

Understanding YAML Syntax

Quick Summary: YAML is indentation-based (Python-like), spaces matter, maps use key: value, lists use dashes, and comments start with #.

Why YAML?

  • Human-readable (not JSON)
  • Less syntax noise (no braces/brackets)
  • Standard for config files and pipelines

Basic Syntax

Comments

# This is a comment
# YAML ignores lines starting with #

Key-Value Pairs (Maps)

name: Rajesh
age: 10
location: Seattle

Lists

languages:
  - C#
  - PowerShell
  - JavaScript
  - Bicep

Nested Objects

person:
  name: Rajesh Dammala
  job:
    title: DevOps Engineer
    team: SupportServices
    level: Level 65

Common Mistakes

MistakeResult
Mixed spaces & tabsParse error
Inconsistent indentationWrong nesting
No space after colonname:value won't parse
Using tabsMost parsers reject

Pipeline Example

trigger:
  - main

pool:
  vmImage: windows-latest

stages:
- stage: Build
  jobs:
  - job: BuildJob
    steps:
    - script: dotnet build
      displayName: Build Solution

Interview Answer

Q: "How do you write YAML for pipelines?"

YAML is indentation-based, so consistency matters. Use spaces (not tabs). Maps use key: value. Lists use dashes. Pipelines define stages → jobs → steps. Our pipeline has Build (compile), Test (MSTest), and Deploy (Bicep) stages. Each step is a script or predefined task. YAML is easier to read and version-control than clicking through UI.
